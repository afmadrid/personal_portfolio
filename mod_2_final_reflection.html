Networks and information Security Management – Final Reflection.


During this module the topic networks and information security management was reviewed. Information security management can be defined as discipline that not only manages technologies, but also people and processes. To ensure security people and processes should be adapted and educated to security measures. Understanding fundamental concepts of networks and their history is important because personal data that travels through the internet uses them. Confidentiality, Integrity, and Availability (CIA) are important components that any secure system must preserve and protect (Campbell, 2016). Confidentiality refers that the information should not be available for unauthorized individuals or organisations. Integrity means that the information delivered must be accurate and complete. Availability means that the information should be available whenever it is requested. A four element was discussed in the module named non-Repudiation which must warranty that traceability to an action or event cannot be negated.

As demonstrated during the analysis of the case of study about how a medical mannequin was compromised, today more than ever security measures should be implemented because not only big networks and financial companies are affected by cyberthreats. Medical devices can be considered as IoT devices that connect with other devices or networks. Due to its limitations of storage and processing power they could be more vulnerable, adding the facts that fewer standard practices exist to build hardware and software for these types of devices. In the medical industry if a medical device is compromised personal data of patients and personal can be disclosed to unauthorized organisations or individuals.

One interesting topic reviewed during the module was the history behind the adoption of the TCP/IP standard rather than the OSI model. Some of the main success factors for the adoption of the TCP/IP standard was the flexibility and rapid adaptability to changes. As discussed on the article “rough consensus and running code” even if the model OSI was proposed by ISO and was a more descriptive representation of a network system. The slow time it took to be implemented and bureaucracy surrounding it make people to move forward to a faster paced standard (Russell, 2006).

GDPR regulations were discussed by analysing cases of study where solutions were not easy to identify and propose. These regulations were stablished in Europe in 2018 and were mandated to be followed with the intention of protecting personal data. Right of access and security by design are concepts described on the GDPR acts. The former refers that anyone should have the right to access its personal information that has been recollected by an organisation (GDPR (a), 2022) and the last refers to the implementation security requirements in processes and products from the beginning through the entire duration of its development life cycle (GDPR (b), 2022).

Definitely one of the most practical and useful topics to understand the responsibilities of a security expert was the scanning exercise performed as part of the final project. For scanning vulnerabilities on the assigned website 5 tools were chosen and the Linux partition Kali Linux was utilised. This Linux partition offered the advantage that it already had installed built-in tools for cybersecurity scanning, and it could be used as a virtualized environment with just the use of a hypervisor (Bhatt, 2018). On the first part of this project security regulations used by the web site and probable vulnerabilities to be found were hypothesised. In the second part next tools were used to perform the scan on the website:

  • Owasp Zap. This tool performed a very complete automated scan where multiple http requests were sent detecting all the components that conformed the website and presented a list of found vulnerabilities categorizing them by severity using different colours. Additional information such as vulnerability description and suggested mitigation actions were presented. The automated scan with this tool last for around 30 minutes and it was noticed that the router IP address was blocked by the targeted website firewall. Using a VPN was a workaround used to continue scanning exercises with other tools.
  • Burp Suite was used to perform a passive scan a community version of this tool. The passive scan allowed to configure the burp suite tool as a proxy and intercept all the requests done by the website. Every time an element or link was clicked on the website folders and files were registered representing the folders and files structures of the web site.
  • The third tool Skipfish started an automated scan and was manually stopped after 10 minutes. With this scan multiple vulnerabilities were found, and a html report was generated allowing a better analysis of the results.
  • Nikto was another tool used which performed a simple scan in the website lasting for no more than 10 minutes. After finishing a list of vulnerabilities found was displayed.
  • Finally, the last scanning tools used was Nmap, this a commonly known scanning tool because it gives information about the opened ports found for the website and the approximated operating system on which the website server is running.

After the scans were performed it was possible to compare the vulnerabilities found on the website by each of the tools. Most of the vulnerabilities were categorized with medium, low and very low severity. One medium categorized vulnerability was the JSLibrary vulnerability which referred as the use of an outdated version of this library on one of the website components a mitigation strategy proposed is improving practices that would guaranty that used libraries and software components are constantly updated or even stop using JavaScript which is considered as an insecure language. Other lower severity vulnerabilities such as the Absence of anti-CSRF tokens, Cookie no HTTP Only Flag, Cookie without same site attribute were proposed to be mitigated by following better practices when building the website.
